APIs Backend de Martin

APIs Get basicas

Estas APIs son un getAll() para cada una de las respectivas tablas de la base de datos, retornan en forma de objetos todos los elementos.

APIs Post basicas

Estas APIs son una simulación de post de paciente, box, y funcionario, el caso del post agendamiento, cambia un poco, debido a que en este caso hay que entregarle parametros, los cuales serian la fecha de inicio y la fecha final de la simulación, las fechas predeterminadas en este caso son '2025-08-19' y '2025-08-20', de modo que por cada bloque de 30 minutos se generara un agendamiento y un estadoagendamiento asociado, el estado sera 'En revisión'.

API para eliminación de datos

En este caso solo existe una API, la cual es para eliminar un agendamiento, esta necesita un parametro, el cual por defecto, y para evitar errores es '-1' en esta guía, cambiarlo a necesidad. Usar con precaución. Esta API, además de eliminar el agendamiento elimina automaticamente el estadoagendamiento asociado, no elimina el resultadosconsulta asociado, debido a que no habria motivo para eliminar un agendamiento si ya se realizo.

APIs para actualizar datos

Estas APIs son para actualizar el estado del agendamiento, la primera solo actualiza el estado y recibe los parametros idagendamiento que predeterminado es '1' y nuevoEstado que de predeterminado es 'Aprobado', luego tenemos la segunda API, la cual solo acepta un parametro el cual es idagendamiento, predeterminado es '1', esta ultima actualiza el estado a 'Aprobado', enseguida, y luego simula los resultadosConsulta.

Es importante aclarar que la primera API solo acepta los siguientes 3 estados 'Aprobado', 'En revisión', y 'Rechazado', cualquier otro estado sera ignorado, y los estados deben ser ingresados de exactamente la misma manera que se presentan en esta guía.

APIs de posteo especiales

La API de posteo personalizado hay que manejarla con cuidado, es por esto, que a pesar de que la api posee 6 parametros, no le di ningun parametro predeterminado, los parametros que recibe son los siguientes en el mismo orden que nombrare, idbox, idfuncionario, idpaciente, fecha, hora de entrada, y hora de salida.

APIs sobre ocupancia de boxes

La Primera API, "Información sobre box ahora" posee 3 parametros de entrada el idbox, la fecha, y la hora. Esta API retorna el id del box, el porcentaje de ocupancia del box, en el día (agendamiento), la especialidad del box, si esta libre u ocupado en este momento, y el proximo bloque agendado.

Pequeña ayuda extra

Algunos de los formatos que se piden son los siguientes:

  • fecha : 'YYYY-MM-DD'
  • hora : 'HH:MM:SS'
  • id : '-1'
  • string : 'cualquierString'
